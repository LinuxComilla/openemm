Online-Hilfe: Einstellungen - Felder verwalten - Neues Feld
Über diesen Eingabedialog erzeugen Sie ein neues Feld für die Profildatenbank. Damit erweitern Sie das Profil aller Empfänger, so dass Sie für jeden einen individuellen Wert in diesem Feld speichern können. Wenn Sie mit Ihren Eingabe fertig sind, klicken Sie auf den Button Speichern. Wollen Sie den Eingabedialog ohne Aktion wieder verlassen, Sie benutzen Sie den Button Abbrechen. Sie gelangen damit zur Übersicht der selbst definierten Felder. Die Eingabefelder und Auswahllisten haben folgende Bedeutung:
  • Feldname: Den Namen zeigt der OpenEMM später immer dann an, wenn Sie ein Feld auswählen können. Sie sollten also einen möglichst eindeutigen Namen verwenden. Sie dürfen alle Zeichen verwenden, sollten den Feldnamen aber nicht zu lange wählen.
  • Beschreibung: In der Beschreibung erläutern Sie den Zweck des Feldes, so dass später auch andere Benutzer wissen, was sie damit anfangen sollen (oder können).
  • Feldname in DB: Dieser Name erscheint nur bei der Verwaltung des Feldes, sonst zeigt ihn der OpenEMM nie an. Es hat sich bewährt, einen zweiteiligen Namen mit einem Unterstrich ( _ ) als Trennzeichen zu verwenden. Denken Sie beispielsweise an Interesse_Autos oder Alter_Jahre. Damit vermeiden Sie Kollisionen mit internen Feldnamen des OpenEMM. Groß- oder Kleinschreibung spielt keine Rolle, denn der OpenEMM wandelt sowieso alles in Großbuchstaben um.
  • Typ: Ein Feld kann unterschiedliche Arten von Daten aufnehmen. Je nach Verwendungszweck wählen Sie den passenden Typ aus. Es stehen folgende Feldtypen zur Verfügung:
    • Datum: In Feldern dieses Typs speichern Sie ein Datum. Der Einsatzbereich von Datumsfeldern ist klar, Sie können damit etwa Geburtstage oder den Ablauftag eines Vertrags speichern. Die Eingabe erfolgt immer nach dem Schema JJJJMMTT, also beispielsweise 19650905 für den 5. September 1965.
    • Numerisch: Der Feldtyp erlaubt als Feldinhalt "nur" Zahlen. Trotz dieser Einschränkung ist der Typ universell. Sie können zum Beispiel speichern, ob sich ein Empfänger für ein bestimmtes Thema interessiert. Ein Wert von 0 bedeutet dann Desinteresse, eine 1 Interesse. Über eine Aktion ist es auch möglich, ein numerisches Feld hochzuzählen (etwa plus 1 bei jeder Reaktion des Empfängers).
    • Alphanumerisch: Bei diesem Feldtyp dürfen Sie beliebige Zeichen im Feld speichern. Verwenden Sie den Typ alphanumerisch aber nur, wenn Sie tatsächlich freie Eingaben ohne Beschränkung erlauben wollen. In einem solchen Textfeld können Sie zum Beispiel Rufnamen speichern oder Urlaubsorte. Beachten Sie aber, dass sich Textfelder oft nur schwer mit der Suchfunktion oder durch Zielgruppen auswerten lassen. Denn es kommt leicht zu unterschiedlichen Schreibweisen, wenn sich ein Anwender beispielsweise bei der Eingabe vertippt.
  • Länge: Das Eingabefeld legt die maximale Anzahl Zeichen fest, die Sie in das neue Feld eintragen können. Das betrifft nur Felder vom Typ Alphanumerisch, die beliebige Zeichen aufnehmen dürfen. Das System legt für Felder vom Typ Date eine Länge von 7 Zeichen fest, für Number eine Länge von 22 Zeichen - andere Werte ignoriert der OpenEMM. Da Alphanumerisch keine Standardlänge hat, müssen Sie unbedingt eine Länge angeben - sonst reagiert das System mit einer Fehlermeldung.
  • Standard-Wert: Das Eingabefeld legt fest, welchen Wert das neue Feld anfangs hat. Wenn Sie beispielsweise einen neuen Empfänger anlegen, trägt der OpenEMM den Standard-Wert in das Profil ein. Für numerische Felder bietet sich also als Standard-Wert 0 an. Wenn es sich zum Beispiel um ein Alphanumerisches-Feld mit der Angabe zum Hauptreiseland handelt, ist als Standard-Wert etwa keines sinnvoll. Dann können Sie zum Beispiel nach einigen Monaten über die Suchfunktion leicht Empfänger herausfiltern, die bisher noch keine Angabe gemacht haben.
    Bitte beachten Sie: Bei Eingaben für das Format Date ist kein Standardwert vorgesehen.
Bitte beachten Sie: Wenn ein Feld erst einmal erzeugt ist, können Sie nur noch Feldname und Beschreibung ändern. Die restlichen Parameter (Länge, Typ etc.) lassen sich nicht mehr anpassen. Der einzige Ausweg ist, das Feld zu löschen und ein neues Feld anzulegen.
