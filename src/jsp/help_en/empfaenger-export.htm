Recipient - Export

The navigation bar offers an Export function which is accessed by the Recipients tab. The Interface is separated in three areas. In the first area you choose which recipient will be exported. You may differentiate between mailing list, recipient type, recipient status and targetgroup. You may use all criteria at the same time, OpenEMM exports all recipients, who meet the criteria.

  • Mailing list: All mailing lists which you have created will be displayed. You can export the particular list by clicking on the corresponding check box.
  • Recipient type: Only the recipient from the chosen types in the mailing list will be exported. You can choose between the following recipient types:
    • All: Without any limitation
    • Normal recipient: All recipients who should receive a newsletter or specified mailing are named as normal recipients.
    • Administrator: A person who has to check all mailings before they get delivered is called Administrator. Only Administrators will be exported by selecting this type.
    • Test recipient:: If you want to test your mailings before deliver, its inevitable to send sample E-Mails to test recipients from the mailing list. These recipients will be exported by selecting this type.
  • Recipient status:Only the recipients in the mailing list with the selected status will be exported. There are five settings available:
    • All: Without any limitation
    • Active: The recipient subscribed to the specified mailing list. You can see under 'note' why the recipient got this status.
    • Bounced: The address is invalid, in vacation-mode or the recipient signed off.
    • Opt-Out by admin:An Admin unsubscribed the recipient
    • Opt-Out by recipient:The recipient unsubscribed himself
  • Target group: Only the recipients will be exported, who are set in the specified target group.

The second area specifies the fields which will be exported with every single recipient. By marking the particular fields every field will create his own column in the final csv-file. You can also choose if you want to see, if the recipients have subscribed/unsubscribed any listed mailing list.
The third area specifies the export file-format. You will always get a textfile, which will be changed in a regular excel format by importing the CSV-file in excel. The unique dropdown menus have the following meaning

  • Separator: If you store recipient data in a spreadsheet software such as Excel and save them in a CSV file, data is usually separated by a semicolon. Other programs may use different separators. You can, however, specify a different separator in the drop-down list.
  • Text recognition character: In case your recipient data contain the separator specified, it must be identified by a different character. If the separator, for instance, is a comma, and if recipient data also contain commas, the commas used as separators must be identified separately before import.
  • Character set:The default value is the Western European character set ISO 8859-1, which you might as well use. You should only select a different character set if your CSV file also uses that character set.
