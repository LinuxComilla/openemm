Recipient - Details

The database contains specific data on each recipient , i.e. his or her profile. You can call up individual recipient data by clicking on the Edit icon or on the relevant E-Mail address. This calls up an entry dialog with the data saved for that particular recipient, under the Recipients tab in the contents area. Data from the profile are organised in three distinct areas. At the top you will find general information, followed by additional profile data and available mailing lists. The marked mailing lists are subscribed by the recipient. By clicking the cancel-button you return to the overview without any changes.

  • Salutation: Sets the recipients gender. The exact salutation can be changed in the mailings.
  • TITLE: title of the receiver (Dr etc.).
  • FIRSTNAME: first name title of the receiver
  • LASTNAME: last name title of the receiver
  • E-Mail: Shows the E-Mail address of the receiver
  • Mail-Format:
    • Text: In Text format, all E-Mail-clients are able to open this mailing, but format instructions (bold, Hyperlinks, Font size ...) will not be included.
    • HTML: In HTML format, images and specific formatting may be added to the mailing. By using images from external AGNITAS-servers, your client may get problems with some browser interfaces using for example a GMX-mail account.
    • Offline-HTML: In Offline-HTML format, images are actually integrated into the E-Mail as a attachments, while in the normal HTML version images are downloaded from the server once the E-Mail is opened.
  • More profile data: The Other profile data section contains fields for internal use as well as all user-defined fields for the recipients’ profiles. Fields for internal use are:
    • change_date: Date of last profile change
    • creation_date: Date of recipient profile creation
    • datasource_id: where the data were imported from User-defined fields follow. This list can be of any length and is based solely on your needs and requirement criteria, for instance customer IDs or other reference numbers.

The sub-menu Mailing Lists contains all mailing lists created so far. The OpenEMM system shows which lists your recipient has subscribed to by ticking them. If the recipient has subscribed to a particular mailing list, the OpenEMM shows additional data.

  • E-Mail: if this option is marked, the recipient has subscribed the mailing list.
  • Type:
    • Administrator: The creator of a mailing is the administrator. You should create a recipient profile for yourself (if you have not already done so) and assign yourself the status of Administrator for that mailing list.
    • Test recipients: To test the mailing, it is essential to send a few test mails to test recipients. They are not real recipients but E-Mail addresses only you have access to. You could also assign that status to someone who has to check a mailing prior to sending it to all recipients on the list.
    • Normal recipient: All recipients who have subscribed to a newsletter and want to receive it are normal recipients.
  • Status: Under Status the OpenEMM shows the recipient's status. The Remark contains a date, a time and the last action.
    • Aktiv:The recipient has subscribed to this mailing list. The remark explains which action caused this status.
    • Bounce:The address is invalid. The recipient was de-registered from the system.
    • Opt-In: The recipient was registered as a subscriber of this mailing list. This may have been done by the recipient himself via the web interface or by the administrator (followed by: by ADMIN).
  • Opt-Out: The recipient has opted out from the mailing list. This may have been done by the recipient himself via the web interface or by the administrator (followed by: by ADMIN).
